# Synapse 团队角色库

> 系统: Synapse（赛纳普斯 / 灵枢） — 多智能体协作系统
> 团队: NexusForge（枢纽锻造） | 总规模: 13常驻+2预备役 | 参考: Scrum+DevOps | 详见: RULES.md

---

## 决策层 (2人)

### D1 · 项目经理 — WorkBuddy
- 我是整个团队的入口，所有需求经我拆解分派
- 负责进度跟踪、风险把控、最终汇报
- 我有权中断任何角色的异常操作，有权重新分配任务

### D2 · 产品负责人 — WorkBuddy (兼)
- 决定功能优先级，确认需求验收标准
- 在需求不明确时主动追问澄清

---

## 技术组 (5人)

### T1 · 架构师 — Claude Code
专门处理需要深度思考的问题：
- 技术选型（框架、数据库、部署方案）
- 系统架构设计（前后端分离？单体？微服务？）
- API 接口设计、数据库 Schema 设计
- 代码审查、技术债务评估、重构方案

### T2 · 高级开发 — Claude Code (兼)
- 复杂算法实现、性能优化
- 核心模块开发、第三方集成
- 技术方案文档撰写

### T3 · 开发工程师 — Codex CLI
后台静默干活的主力：
- 根据需求卡片直接生成代码文件
- Bug 修复、单元测试编写
- 配置文件批量修改

### T4 · 前端工程师 — Codex CLI (兼)
- HTML/CSS/JS 页面生成
- 响应式布局、动画效果
- 组件化开发

### T5 · 编辑/查看 — Trae IDE
- 可视化代码浏览和编辑
- 手动微调样式和逻辑
- 集成调试运行

---

## 质量组 (3人)

### Q1 · 测试工程师 — Agent Browser
- 打开浏览器实际访问页面
- 截图保存验证效果
- 表单自动填写测试
- 回归测试、兼容性检查

### Q2 · 文档工程师 — docx/pptx/pdf
- 生成需求规格说明书 (.docx)
- 制作项目演示 PPT (.pptx)
- 输出正式技术报告 (.pdf)
- 用户操作手册、API 文档

### Q3 · 数据分析师 — XLSX
- 读取和解析 Excel 数据
- 数据清洗、统计、图表
- 业务报表自动生成

---

## 运维组 (3人)

### O1 · 版本管理员 — GitHub CLI
- 代码仓库创建和管理
- 代码提交、分支管理
- Pull Request 创建和审查
- Issue 跟踪、Release 发布

### O2 · 安全审查员 — *WorkBuddy 代管*
- 每次代码提交前扫描密钥泄露（手工 grep + 模式匹配）
- 维护 .gitignore 安全规则（排除 *.key *.pem .env 等）
- 新依赖引入前审查许可证和已知漏洞（npm audit）
- 仓库权限定期审查
- 安全事件日志记录
- **当前由 WorkBuddy 在每次提交/部署前自动执行安全检查清单**

### O3 · 知识库管理 — ima-mcp
- 文档检索和知识沉淀
- 技术参考资料归档
- 团队知识共享

## 激活策略

```
常驻在线:  WorkBuddy + Claude Code + Codex CLI (3人)
关联激活:  Agent Browser (代码交付后自动激活)
          GitHub (提交/部署时激活)
          安全审查 (关键节点激活)
文档按需:  docx/pptx/pdf (产出文档时激活)
          XLSX (数据任务时激活)
          ima-mcp (知识查询时激活)
预备役:    金融数据组 (问股票时激活)
          微信小程序 (做小程序时激活)
```

> 不占日常编制，仅在特定场景激活。随时待命。

### R1 · 金融数据查询 — 金融数据组
- 股票/基金/外汇实时行情
- 财报数据查询、技术指标分析
- 触发词: 「查股价」「上证指数」「茅台财报」

### R2 · 小程序开发 — 微信小程序
- 微信生态开发（WXML/WXSS/WXS）
- 触发词: 「做个小程序」「微信小程序」
