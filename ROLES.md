# 团队角色分配

> 最后更新: 2026-06-09  
> 总成员: 11 人

---

## 核心层

### 💜 WorkBuddy — 项目经理 / 总调度
- **职责**: 需求拆解、任务分配、状态汇报、工具编排
- **工作方式**: 接收用户需求 → 拆分为子任务 → 路由给对应成员 → 汇总结果 → 汇报
- **触发**: 所有任务的第一入口

### 💙 Claude Code — 架构师
- **职责**: 技术选型、系统设计、代码审查、架构决策
- **工作方式**: 深度思考型任务，交互式对话
- **触发**: 「设计架构」「技术选型」「审查代码」「重构方案」

### 💚 Codex CLI — 程序员
- **职责**: 代码实现、文件生成、测试编写、Bug 修复
- **工作方式**: 静默后台执行，输出代码文件
- **触发**: 「写个函数」「生成页面」「修复 Bug」「跑测试」

### 🧡 Trae IDE — 编辑器 / 查看器
- **职责**: 可视化代码浏览、手动微调、集成调试
- **工作方式**: 桌面 IDE，用户手动打开查看产物
- **触发**: 最终代码产物查看

---

## 专业层

### 🧪 Agent Browser — 测试员
- **职责**: 浏览器自动化测试、页面截图验证、表单填写
- **工作方式**: 启动 Chromium 访问页面 → 截图 → 交互验证
- **触发**: 「测试一下」「打开看看」「截图」「填表单」
- **状态**: ⏳ 安装中

### 📄 文档组 (docx/pptx/pdf) — 文档员
- **职责**: 生成 Word 文档、PPT 演示、PDF 报告
- **工作方式**: 按模板生成正式文档
- **触发**: 「生成需求文档」「做个 PPT」「导出 PDF」

### 📊 XLSX 工具 — 数据员
- **职责**: Excel 数据读取、分析、报表生成
- **工作方式**: 解析电子表格、执行公式、生成图表
- **触发**: 「处理这个 Excel」「生成数据报表」

### 🔍 GitHub 连接器 — 审查员
- **职责**: 代码推送到仓库、PR 审查、Issue 管理
- **工作方式**: CLI 操作 GitHub API
- **触发**: 「推到 GitHub」「提 PR」「看 Issue」
- **状态**: ⚠️ 待连接

### 🚀 EdgeOne Pages — 部署员
- **职责**: 一键部署静态网站到公网 CDN
- **工作方式**: 上传文件 → 返回公网 URL
- **触发**: 「部署上线」「发布网站」
- **状态**: ⚠️ 待连接

### 💹 金融数据组 — 行情员
- **职责**: 股票、基金、财报数据实时查询
- **工作方式**: 自然语言查询金融数据
- **触发**: 「查一下茅台股价」「上证指数多少」

## 工作流示例

```
用户: "帮我做一个在线计算器，测试通过后部署上线"

WorkBuddy 拆解:
  ├── 架构设计 → Claude Code (技术选型: 纯 HTML/JS? React?)
  ├── 代码实现 → Codex (生成 calculator.html)
  ├── 功能测试 → Agent Browser (打开页面截图验证)
  ├── 生成文档 → docx (写用户手册)
  ├── 推送到 Git → GitHub (commit + push)
  └── 部署上线 → EdgeOne Pages (发布到公网)

最终汇报 → 用户在 Trae 中查看代码 → 公网链接可直接打开
```
