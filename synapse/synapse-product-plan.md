## 1. 赛道分析 & 定位

### 1.1 市场空白

当前 AI 内容创作市场分三层，**没有一个人把三层串起来**：

```
层3: 多平台分发    聚媒通 / 新榜小豆芽 / Postudio  ← 只管发，不管写
层2: AI 内容生成    Jasper / 豆包 / ChatGPT           ← 只管写，不管品牌统一
层1: 语言打磨       秘塔写作猫                          ← 只管改，不管创作
```

**6 家竞品的致命缺陷**：

| 竞品 | 致命缺陷 | Synapse 突破口 |
|------|---------|---------------|
| 秘塔写作猫 | 不会"创作"，只能"润色" | Brand Brain 驱动从零创作 |
| Jasper | 中文差 + 贵($49) + 无分发 | 中文优先 + 独立创作者定价 |
| Notion AI | 不是内容工具 | 内容创作专用工作流 |
| Postudio | Alpha 阶段，只发不写 | 创作+分发一体 |
| 聚力GEO | 品牌管理弱 | Brand Brain 作为核心差异化 |
| 课件帮 | 只管做课件，不管选题和分发 | 选题→课件→分发→数据闭环 |

### 1.2 Synapse 核心差异化（4 个护城河）

1. **Brand Brain 共享记忆层** — Jasper 有但中文差，别人完全没有
2. **创作+分发一体** — 所有竞品都是二选一
3. **双通道（文字+视频）** — 竞品全都没有
4. **数据闭环飞轮** — 监控→分析→优化→再生，市场绝对空白

### 1.3 对标总结

每做一个模块，先问三个问题：
- 这个模块的最佳竞品是谁？他们怎么做？—— **借鉴**
- 他们哪里做得不好？用户抱怨什么？—— **改良**
- 有没有他们没做的事？—— **创新**

## 3. 模块详解 & 竞品对标

### M1 — 文案收集对标

**竞品对标**：

| 竞品 | 月费 | 做什么 | 缺陷 |
|------|------|--------|------|
| 新榜 | ¥500-2000 | 内容榜单+趋势数据 | 贵、只查不写 |
| 蝉妈妈 | ¥500+ | 抖音带货数据分析 | 偏带货、不跨平台 |
| 灰豚数据 | ¥300+ | 多平台数据 | 数据多但不生成 |

**借鉴 + 改良**：
- 借鉴：新榜的内容榜单逻辑、蝉妈妈的爆款提取
- 改良：加上 AI 自动拆解爆款文案结构 → 提取话术模板 → 存入 Brand Brain
- 创新：收集→学习→生成 一条线，不需要人在中间手动操作

**真实测试方法**：
- 设定一个品类（如"AI 工具推荐"）
- 自动抓取该品类小红书/抖音 Top 50 爆款文案
- AI 分析提取：标题结构/开头话术/卖点排列/结尾CTA
- 生成 5 篇自创文案 → 发布到真实账号
- 对比原创 vs 竞品爆款的互动数据
- 判断标准：原创文案点赞率 ≥ 竞品平均水平的 70% = 及格

### M2 — 多平台文案发布

**竞品对标**：

| 竞品 | 月费 | 平台数 | 缺陷 |
|------|------|--------|------|
| 聚媒通 | ¥299-599 | 13+ | 只管发不管写，无 AI 适配 |
| 新榜小豆芽 | 免费-¥199 | 20 | 格式适配弱，品牌一致性无 |
| 蚁小二 | ¥99+ | 多平台 | 纯工具，无内容能力 |
| 融媒宝 | ¥99-299 | 多平台 | 偏企业，创作者友好度低 |

**借鉴 + 改良**：
- 借鉴：聚媒通的多平台 API 对接、小豆芽的一键分发
- 改良：加上 AI 自动适配格式——同一主题自动生成小红书版(emoji+标签+封面)、微信版(深度长文)、抖音版(短文案+话题)
- 创新：Brand Brain 注入 → 确保每个平台发的都是"你"的声音

**真实测试方法**：
- 用 M1 生成的 5 篇文案
- 通过 M2 自动适配为 3 平台版本并发布
- 追踪 48 小时内各平台互动数据
- 对比：同一内容三平台版本的表现差异
- 优化：哪个平台的适配模板效果最好 → 调整模板规则

### M3 — 文案→视频生成

**竞品对标**：

| 竞品 | 月费 | 能力 | 缺陷 |
|------|------|------|------|
| 可灵(Kuaishou) | 免费-会员 | 文生视频/图生视频 | 只管生成不管分发 |
| 即梦AI(字节) | 免费-会员 | 文生视频 | 无品牌一致性 |
| 通义万相(阿里) | 免费 | 文生视频 | 功能单一 |
| Runway | $15/月 | 文生视频/编辑 | 英文优先 |

**借鉴 + 改良**：
- 借鉴：可灵/即梦的视频生成 API
- 改良：加上 Brand Brain 注入品牌视觉风格（颜色/字体/Logo/片头片尾）
- 创新：M2 文案→M3 自动接收→生成品牌风格一致视频→M2 发布（一条线无需手动）

**真实测试方法**：
- 取 M1 数据：对比同一主题下，纯文案 vs 文案+视频的播放/互动差异
- 测试 2 种视频风格：真人出镜素材 vs AI 生成素材
- 追踪完播率和互动率
- 判断标准：有视频的帖子播放量 > 纯文案的 2x = 及格

### M4 — 全域数据监控

**竞品对标**：

| 竞品 | 缺陷 |
|------|------|
| 各平台原生后台（抖音创作者中心、微信公众号后台等） | 数据分散——抖音一个、微信一个、小红书又一个 |
| 新榜/蝉妈妈（企业级） | 个人创作者买不起 |

**借鉴 + 改良**：
- 借鉴：各平台的数据指标定义（播放量/点赞/评论/转发/完播率）
- 改良：聚合到统一仪表盘 + 自动计算内容 ROI（投入时间 vs 产出流量）
- 创新：异常提醒——某条内容突然爆了/突然降了 → 推送通知

**真实测试方法**：
- 对接 3 个平台 API（小红书、抖音、微信公众号）
- 拉取用户最近 30 天数据
- 计算每条内容 ROI：发布耗时(分钟) vs 播放量/互动数
- 识别：哪些内容 ROI 最高 → 标记为"爆款基因"
- 验证：ROI 计算与实际感受是否一致

### M5 — 智能互动引擎

**竞品对标**：

| 竞品 | 缺陷 |
|------|------|
| 科握（企业级社媒互动） | 贵，不适合个人创作者 |
| 各平台原生评论管理 | 分平台操作，手动回复效率低 |

**借鉴 + 改良**：
- 借鉴：科握的消息聚合逻辑
- 改良：加上 AI 风格化回复——你可以设定"幽默/专业/温暖"等风格
- 创新：智能优先级排序——哪些评论值得优先回复？哪些是负面需要紧急处理？

**真实测试方法**：
- 手动回复的评论 vs AI 建议回复的评论
- 对比回复后的用户二次互动率（回了之后用户会不会再互动）
- 对比人工回复耗时 vs AI 辅助回复耗时
- 判断标准：AI 回复的二次互动率 ≥ 人工回复的 80% = 及格

### M6 — 智能投流引擎

**竞品对标**：

| 竞品 | 缺陷 |
|------|------|
| DOU+（抖音）、薯条（小红书）、微信广告 | 全手动判断投不投、投多少 |
| 无跨平台预算分配工具 | 创作者凭感觉分配预算 |

**借鉴 + 改良**：
- 借鉴：各平台的原生投放 API
- 改良：设定预算上限 → AI 根据实时 ROI 自动分配
- 创新：低 ROI 内容自动停投 + 高 ROI 自动加投 → 跨平台动态分配

**真实测试方法**：
- 设定 ¥200 总预算，分配 3 条内容到 2 个平台
- AI 自动分配 vs 手动平均分配
- 对比 48 小时后的新增粉丝/互动/转化
- 判断标准：AI 分配组的 ROI ≥ 手动组的 1.2x = 及格

### M7 — 电商带货集成

**竞品对标**：

| 竞品 | 缺陷 |
|------|------|
| 抖音小店/小红书店铺/微信小商店 | 分平台管理，各管各的 |
| 内容创作和商品管理完全割裂 | 写文案在一个工具，挂商品在另一个 |

**借鉴 + 改良**：
- 借鉴：各平台的商品 API
- 改良：统一商品库（一次录入，跨平台用）
- 创新：内容创建时自动匹配推荐商品 + 追踪"内容→商品点击→成交"完整链路

**真实测试方法**：
- 选 5 个商品，写 5 篇带货文案
- 对比：有商品推荐的文案 vs 无商品推荐的文案 → 点击率和转化率
- 追踪从文案到成交的完整转化漏斗
- 判断标准：转化率 > 0.5% = 有商业价值

### M9 — 数据闭环飞轮（核心差异化，市场绝对空白）

**竞品对标**：无——整个行业没有任何竞品做到这一点。

**独有创新**：

```
监控数据(M4) → AI分析爆款基因 → 自动生成新内容(M1/M2/M3)
                                      ↓
        ← 新数据反馈(M4) ← 发布(M2) ←─┘
```

- 不是你手动"看看数据然后想下一期做什么"
- 是 AI 自动判断"这个话题+这个结构+这个发布时间=爆款"
- 然后 AI 自动生成类似结构的新内容
- 你只需要确认发布

**真实测试方法（关键验证）**：
- Phase 4 开始做闭环
- 让飞轮跑 2 周（完全 AI 驱动，你只确认发布）
- 对比：飞轮前 2 周 vs 飞轮后 2 周的总体流量/粉丝增长/互动
- 判断标准：飞轮启动后自然流量增长 ≥ 30% = 成功
- 同时监控：飞轮会不会导致内容同质化 → 如果出现，加入"多样性注入"机制

**竞品对标**：

| 竞品 | 月费 | 优势 | 缺陷 |
|------|------|------|------|
| 课件帮 | ¥68 | PPT→视频一体，智能出题 | 无选题发现，无分发 |

| 课灵 | 万元+ | 99%数字人还原 | 企业定价，独立创作者用不起 |

**借鉴 + 改良**：

从课件帮借鉴：PPT→视频一键转换流程、智能出题、长文本解析
从 HeyGen 借鉴：语音克隆、数字人质量分级(标准/4K)
从 Synthesia 借鉴：Brand Kit 品牌工具包
从腾讯智影借鉴：PPT 模式数字人(PPT页+AI讲解)

**4 个独有创新**：
1. 自动选题 — M4 监控全网爆款课程主题 → AI 推荐高需求低竞争方向
2. 内容→课程桥 — M1/M2 已写爆款文案自动转化为课程脚本
3. 多平台分发 — 完整课程→B站/小鹅通，切片→小红书/抖音/视频号
4. 完课率追踪 — 追踪完课率，AI 自动优化低完课率章节

**真实测试方法**：
- 选题验证：用 M1 爬取"AI 工具教程"关键词的搜索量 → 验证是否有真实需求
- 内容验证：把已发布的爆款小红书笔记转化为 5 节微型课程
- 分发验证：完整课程发 B 站，切片发小红书/抖音 → 对比播放量
- 付费验证：前 3 节免费，后 2 节设置为 ¥9.9 付费 → 测试付费转化率
- 判断标准：完课率 > 40% + 付费转化率 > 3% = 商业可行

## 4. 真实流量测试方案

### 4.1 核心原则

> **不先做好整个系统再测 — 每个模块做出来，立刻用真实内容、真实平台、真实数据去验证。**

三不要：
- 不要用模拟数据做测试
- 不要在"开发环境"里判断好坏
- 不要等全部做完再回头看

### 4.2 每个模块的验证指标

| 模块 | 核心指标 | 及格线 | 目标线 |
|------|---------|--------|--------|
| M1 文案收集 | 原创文案点赞率 vs 竞品爆款 | ≥ 70% | ≥ 100% |
| M2 多平台发布 | 三平台版本 48h 互动总量 | > 0 (有互动) | 单篇 > 50 互动 |
| M3 文案→视频 | 有视频的帖子 vs 纯文案播放量 | ≥ 2x | ≥ 5x |
| M4 数据监控 | ROI 排名与实际感受一致度 | 合理 | 自动发现 3 条爆款基因 |
| M5 智能互动 | AI 回复后用户二次互动率 | ≥ 80%(vs人工) | ≥ 100% |
| M6 智能投流 | AI 分配 vs 手动分配 ROI | ≥ 1.2x | ≥ 2x |
| M7 电商带货 | 带货文案转化率 | > 0.5% | > 2% |
| M8 智能客服 | 客服介入后转化率提升 | ≥ 15% | ≥ 30% |
| M9 数据闭环 | 飞轮启动后自然流量增长 | ≥ 30% | ≥ 50% |
| M10 SaaS | 付费用户数 | 月活 100 | 月活 1000 |
| M11 课件引擎 | 完课率 + 付费转化率 | 完课>40% 转化>3% | 完课>60% 转化>10% |

### 4.3 测试节奏

每个 Phase 包含三个子阶段：
1. **Build**（占 40% 时间）：开发模块核心功能
2. **Live Test**（占 40% 时间）：发布到真实平台，追踪真实数据
3. **Iterate**（占 20% 时间）：根据数据反馈调整功能

### 4.4 数据追踪基础设施

每个模块上线前必须接入：
- 内容 ID → 平台 URL → 发布时间 → 48h/7d/30d 播放量/点赞/评论/转发/收藏
- 所有数据写入统一的 `data/metrics.jsonl`，方便后续 M4 读取

## 6. 技术架构

### 6.1 技术栈

| 层 | 技术 | 说明 |
|----|------|------|
| 前端 | HTML/CSS/JS (SPA) | 轻量亮色主题 + 毛玻璃质感，参考 content-engine |
| 后端 | Node.js + Express | 轻量，与现有架构一致 |
| AI 引擎 | DeepSeek API | 文案生成/分析/改写 |
| 视频生成 | 可灵/即梦 API | 选择免费或低价方案 |
| 数字人 | HeyGen API / 腾讯智影 API | Phase 3 评估性价比后决定 |
| 数据存储 | JSON 文件(Phase 1-2) → SQLite(Phase 3+) | 随数据量增长升级 |
| 多平台发布 | 各平台官方 API | 小红书/抖音/微信/视频号/B站 |
| AI 路由 | 已有 AI Router (:3100) | 自动分派任务到最优 AI 通道 |

## 8. 风险 & 应对

| 风险 | 概率 | 影响 | 应对 |
|------|------|------|------|
| 平台 API 封禁 | 中 | 高 | 优先用平台官方 API；爬虫加频率控制和人机验证 |
| AI 生成质量不稳定 | 高 | 中 | 每次生成加人工确认环节；Brand Brain 降低语调漂移 |
| 内容同质化（飞轮副作用） | 中 | 高 | M9 加入多样性注入机制；设定同质化指数阈值预警 |
| DeepSeek API 涨价/限流 | 低 | 高 | 预留备用 API（豆包、通义千问） |
| 数字人 API 成本高 | 中 | 中 | Phase 3 做成本实验对比 HeyGen vs 腾讯智影 vs 万兴 |
| 多平台 API 对接复杂 | 高 | 中 | 先做 3 个核心平台（小红书+微信+抖音），其他逐步加 |
| 用户觉得功能太多太杂 | 中 | 中 | 分模块界面；首次使用引导只展示当前 Phase 功能 |
