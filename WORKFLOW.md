## 核心洞察（来自全网最佳实践）

| 发现 | 我们的调整 |
|------|-----------|
| Codex CLI 终端任务**便宜 10 倍** | 批处理/文件操作 → 优先派 Codex |
| Claude Code **代码质量高 67%** | 架构/审查 → 指定 Claude Code |
| Supervisor 模式最可控 | 保持 WorkBuddy 总调度 |
| 从 2 Agent 起步最优 | 精简核心层，其余按需激活 |
| 规则路由省 Token | 简单任务直接用规则匹配，不动 LLM |
| 终止保护防止死循环 | 加 token 上限 + 最大轮次 |

## 任务路由规则（Rule-First）

### 🟢 规则路由（零 LLM 成本，<1ms）

| 触发词 | → 路由到 | 原因 |
|--------|---------|------|
| 设计/架构/选型/重构/审查 | Claude Code | 需要深度推理 |
| 生成/创建/写/修复/改 | Codex CLI | 实现类任务 |
| 测试/截图/打开看看 | Agent Browser | 测试类任务 |
| 文档/PPT/PDF/Word | docx/pptx/pdf | 文档类任务 |
| 推送/提交/PR/仓库 | GitHub CLI | 版本管理 |
| Excel/数据/报表 | XLSX | 数据类任务 |

### 🟡 LLM 路由（仅在规则无法匹配时）

仅当用户需求无法被上述关键词匹配时，WorkBuddy 才使用 LLM 判断任务类型。

## 成本感知调度

### Token 预算（每任务）

| 任务复杂度 | 预算上限 | 超限处理 |
|-----------|---------|---------|
| 简单（改个文件） | 50K tokens | 汇报后等确认 |
| 中等（新功能） | 200K tokens | 拆分为子任务 |
| 复杂（系统重构） | 500K tokens | 分批执行 |

### 工具选择策略

```
需要代码质量 → Claude Code（贵但准）
需要批量生成 → Codex CLI（便宜 10×）
需要浏览器验证 → Agent Browser
需要文档输出 → docx/pptx/pdf
需要版本管理 → GitHub CLI
```

## 终止保护

| 保护机制 | 设置 |
|---------|------|
| 最大轮次 | 10 轮（防止无限循环） |
| Token 上限 | 500K/任务 |
| 人工检查点 | 高风险操作前确认 |
| 超时保护 | 单 Agent 执行不超过 5 分钟 |

## 标准工作流（优化版）

```
1. 收到需求 → WorkBuddy 关键词路由（规则匹配）
2. 规则命中 → 直接分派，跳过 LLM 决策
3. 执行 → 单 Agent 完成（不嵌套，减少 Token）
4. 检查 → 超过 Token 预算 80% 时预警
5. 交付 → WorkBuddy 汇总，汇报给用户
6. 记录 → 关键决策写入 MEMORY.md
```
