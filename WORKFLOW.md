# 工作流程

## 需求处理流程

```
用户需求
   │
   ▼
WorkBuddy (项目经理) — 拆解写入 CURRENT.md
   │
   ├── 架构/设计 ──▶ Claude Code ──▶ 技术方案
   │
   ├── 代码实现 ──▶ Codex CLI ──▶ 代码文件
   │
   ├── 测试验证 ──▶ Agent Browser ──▶ 截图确认
   │
   ├── 文档产出 ──▶ docx/pptx/pdf ──▶ 正式文档
   │
   ├── 数据报表 ──▶ XLSX ──▶ 数据分析
   │
   ├── 版本管理 ──▶ GitHub ──▶ 仓库推送
   │
   ├── 部署上线 ──▶ EdgeOne Pages ──▶ 公网链接
   │
   └── 金融数据 ──▶ 金融组 ──▶ 实时行情
   │
   ▼
WorkBuddy 汇总 → 汇报结果 → Trae 查看产物
```

## 调度规则

1. **架构/设计任务** → Claude Code
   - 技术选型、API 设计、数据库 Schema、重构方案
   - 交互式深度对话

2. **代码实现任务** → Codex CLI
   - 生成文件、编写测试、配置修改
   - 静默后台 + 输出产物

3. **测试任务** → Agent Browser
   - 打开浏览器访问页面
   - 截图对比、交互验证
   - 产出截图 + 测试报告

4. **文档任务** → docx/pptx/pdf
   - 需求文档 (.docx)
   - 演示文稿 (.pptx)
   - 正式报告 (.pdf)

5. **数据任务** → XLSX
   - 读取 Excel
   - 数据清洗
   - 生成报表

6. **版本任务** → GitHub
   - 推送代码
   - 创建 PR
   - Issue 管理

7. **部署任务** → EdgeOne Pages
   - 静态站点发布
   - 获取公网 URL

8. **金融任务** → 金融数据组
   - 股票行情
   - 基金数据
   - 财报查询

## 汇报格式

每次任务完成后汇报：
- ✅ **完成内容** — 简述做了什么
- 📁 **产物清单** — 文件路径 + 链接
- 🧪 **测试结果** — 截图/通过/失败
- ⚠️ **注意事项** — 已知问题
- 💡 **下一步** — 可用 Trae 查看 + 后续操作
